# Экспорт данных Features Section

## Как это работает

Компонент `FeaturesSection` сохраняет данные в **localStorage** браузера. Для применения изменений в продакшене нужно экспортировать данные в статический JSON файл.

## Экспорт данных в JSON

### Способ 1: Через консоль браузера (рекомендуется)

1. Откройте страницу с компонентом FeaturesSection в режиме разработки
2. Внесите изменения (добавьте изображения, измените текст)
3. Откройте консоль браузера (F12)
4. Выполните команду:
   ```javascript
   exportFeaturesToJSON()
   ```
5. Файл `features.json` будет автоматически скачан
6. Скопируйте содержимое скачанного файла в `app/data/features.json`
7. Закоммитьте изменения и выполните билд проекта

### Способ 2: Вручную

1. Откройте консоль браузера (F12)
2. Выполните:
   ```javascript
   localStorage.getItem('features_section_data')
   ```
3. Скопируйте полученный JSON
4. Вставьте в файл `app/data/features.json` (замените содержимое)
5. Закоммитьте изменения и выполните билд проекта

## Структура данных

Файл `app/data/features.json` должен содержать массив из 3 объектов:

```json
[
  {
    "backgroundImage": null,
    "text": "Для дома",
    "textColor": "#ffffff"
  },
  {
    "backgroundImage": "data:image/jpeg;base64,/9j/4AAQ...",
    "text": "Для офиса",
    "textColor": "#000000"
  },
  {
    "backgroundImage": "cover-bg.jpg",
    "text": "Для сада",
    "textColor": "#ffffff"
  }
]
```

### Формат backgroundImage

- `null` - без изображения
- `"data:image/..."` - base64 изображение (будет использоваться как есть)
- `"cover-bg.jpg"` - ключ из доступных изображений проекта
- `"/uploads/images/..."` - путь к загруженному изображению

## Важные замечания

1. **Base64 изображения** сохраняются в JSON файле и будут включены в билд. Это увеличивает размер файла, но работает на статическом хостинге.

2. **Изображения из проекта** (например, `cover-bg.jpg`) должны существовать в `app/assets/images/`.

3. **Загруженные изображения** сохраняются как base64 в localStorage и в JSON файле.

4. После обновления `app/data/features.json` выполните билд проекта:
   ```bash
   yarn build
   ```

5. В продакшене данные загружаются из JSON файла, если в localStorage нет сохраненных данных.

## Автоматизация (опционально)

Можно создать скрипт для автоматического экспорта перед билдом:

```bash
# В package.json добавить:
"prebuild": "node scripts/export-features.js"
```

Но для этого нужно будет сохранять данные localStorage в файл перед билдом.
